## Story Spec

1. Stories have two types of meta data - functional and user-facing
2. Functional meta data include: tags, story type, chapter type, section type, order,
   chapter limit, visibility, slug and template
3. User-facing metadata include: title, description, cover image, section titles and
   section descriptions
4. Stories can be ordered
5. A story can have sections
6. A translation can't add or remove story sections
7. A story can be published when all its user-facing metadata is translated and all its
   chapters are complete
8. By default, stories can’t be un-published
9. An empty, unpublished story can be deleted
10. All published stories are included in the API
11. A user can select a chapter definition from fixed templates: course, devotion and any
    hard-configured bespoke definitions
12. Once a story is created, the user can not change the template selection or slug
13. Legacy, bespoke stories can not be unpublished
14. A user can select locale filtered resources to attach to a story
15. A user can remove a resource from a story
